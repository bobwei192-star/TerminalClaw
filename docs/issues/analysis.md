TerminalClaw 问题分析文档
==========================

## 一、问题分类框架

### 1.1 问题优先级定义

| 优先级 | 定义 | 响应时间 |
|--------|------|----------|
| P0 | 阻断性问题，核心功能无法使用 | 立即修复 |
| P1 | 重要功能问题，影响用户体验 | 1-3 天 |
| P2 | 次要问题，不影响核心功能 | 1-2 周 |
| P3 | 改进建议，优化性质 | 按需安排 |

### 1.2 问题类型分类

| 类型 | 描述 | 示例 |
|------|------|------|
| 功能缺陷 | 功能无法正常工作 | 日志不写入文件 |
| 性能问题 | 响应慢、资源占用高 | OpenClaw 冷启动 60s |
| 兼容性问题 | 特定环境下无法运行 | Windows 原生不支持 |
| 用户体验 | 使用不方便、不直观 | 右面板无法输入 |
| 安全问题 | 潜在安全风险 | 命令执行无限制 |
| 代码质量 | 代码可维护性问题 | 缺少测试、无类型注解 |

## 二、核心问题深度分析

### 2.1 全自动日志捕获

**问题描述**：用户必须手动添加 `| tee -a /tmp/tclaw/live.log`，否则日志不会被记录。

**根因分析**：
- 当前实现使用 `eval "command" 2>&1 | tee -a live.log` 的方式
- tee 是命令级别的，不是 Shell 级别的
- 用户在左面板执行新命令时，新输出不会自动 tee

**影响评估**：
- 严重影响用户体验（P0）
- 用户需要记住管道语法
- 自然命令无法被分析

**解决方案对比**：

| 方案 | 复杂度 | 效果 | 推荐 |
|------|--------|------|------|
| script 命令 | 中 | 全自动捕获所有输出 | ✅ |
| PROMPT_COMMAND | 高 | 每次命令后追加 | ❌ |
| 自定义 Shell | 高 | 完全控制 | ❌ |

**实施路径**：修改 `tmux_manager.py` 的 `init_log_pane()` 函数，使用 `script -q -f -a live.log -c bash` 启动左面板。

### 2.2 OpenClaw 性能瓶颈

**问题描述**：OpenClaw Agent 冷启动时间超过 60 秒。

**根因分析**：

| 原因 | 占比 | 说明 |
|------|------|------|
| 上下文膨胀 | 40% | 工具返回的日志堆积在对话历史 |
| 工具注入过多 | 25% | 默认加载 browser、cron、message 等重量级工具 |
| 模型冷启动 | 20% | Ollama 默认 5 分钟卸载模型 |
| 二次方复杂度 | 15% | 上下文从 4K 到 8K，计算量是 4 倍 |

**影响评估**：
- 严重影响实时分析体验（P1）
- 用户等待时间过长
- 可能导致用户放弃使用

**解决方案**：组合优化措施，预期将延迟从 60s 降至 8-15s。

### 2.3 右面板交互性

**问题描述**：右面板运行 Live Monitor 循环时无法输入指令。

**根因分析**：
- Live Monitor 是阻塞式的 while True 循环
- Python 进程占用 stdin，用户输入被忽略
- Tmux 不传递按键到阻塞进程

**影响评估**：
- 中等影响用户体验（P1）
- 用户无法在监控时执行交互式分析
- 需要退出监控才能输入

**解决方案对比**：

| 方案 | 复杂度 | 用户体验 | 推荐版本 |
|------|--------|---------|----------|
| 快捷键暂停/恢复 | 中 | ⭐⭐⭐⭐ | v0.2 |
| 改进提示信息 | 低 | ⭐⭐⭐ | v0.1 |
| 多线程非阻塞 | 高 | ⭐⭐⭐⭐⭐ | v0.2 |

### 2.4 可配置触发模式

**问题描述**：当前只检测固定的错误关键词（ERROR/FATAL/EXCEPTION 等）。

**根因分析**：
- 触发模式硬编码在代码中
- 没有提供用户配置接口
- 用户可能需要监控特定业务错误

**影响评估**：
- 影响功能灵活性（P2）
- 无法适应不同场景需求

**解决方案**：
1. 添加 `TCLAW_TRIGGER_PATTERNS` 环境变量
2. 提供预设模式（errors_only/warnings_too/everything）
3. 在 banner 中显示当前模式

### 2.5 跨平台兼容性

**问题描述**：Windows 原生环境不支持。

**根因分析**：
- OpenClaw 在 Windows 原生环境存在 exec 空输出 Bug
- Tmux 是 Unix 工具，Windows 原生不可用
- 日志路径 `/tmp/tclaw` 在 Windows 上不存在

**影响评估**：
- 限制目标用户群（P1）
- Windows 用户必须使用 WSL2

**解决方案**：
1. 文档明确要求 WSL2 环境
2. 配置文件支持 Windows 路径
3. 测试覆盖 WSL2 环境

## 七、结论

### 7.1 当前重点

1. **解决全自动日志捕获**（P0）：使用 script 命令
2. **实现双模式切换**（P0）：支持 API 直连和 OpenClaw
3. **提供可配置触发**（P0）：TCLAW_TRIGGER_PATTERNS

### 7.2 技术债务

1. 测试覆盖率不足
2. 缺少类型注解
3. 错误处理不完善

### 7.3 未来方向

1. 多线程交互式面板
2. 指令库索引化
3. 记忆系统整合
4. 与 rejoin 等工具链联动
