四、Phase 1 — 借鉴项目研究（预计 90 分钟）

4.1 Gonzo（control-theory/gonzo）

    借鉴输出: 记录在 PROBLEMS.md

4.2 Claw Core（@wchklaus97hk/claw-core）
    npm pack @wchklaus97hk/claw-core; tar xf ... 提取源文件

    借鉴输出: 记录在 PROBLEMS.md

4.3 rejoin .jsonl 会话格式
    pip install rejoin 2>/dev/null; 分析源码
    重点研究: ~/.openclaw/agents/**/*.jsonl 解析逻辑
    借鉴输出: 记录在 PROBLEMS.md

五、Phase 2 — 核心实现（预计 300 分钟）

5.1 terminalclaw/config.py
    常量定义: LOG_DIR, LOG_FILE, SESSION_NAME, MAX_LOG_SIZE_MB
    环境变量读取: TCLAW_LOG_DIR, TCLAW_SESSION, TCLAW_MODEL

5.2 terminalclaw/env_check.py（Python 实现）
    check_tmux() → bool
    check_openclaw() → bool
    check_ollama_service() → bool（HTTP 请求 /api/tags）
    check_ollama_model(model_name) → bool
    check_requirements() → List[str]（缺失依赖列表）
    环境自检入口: run_checks() → Dict

5.3 terminalclaw/log_manager.py（Python 实现）
    setup_log_dir(path) → None（创建目录 + touch 文件）
    get_log_info(path) → Dict（size, lines）
    rotate_log(path, max_mb) → bool（超阈值则轮转）
    count_errors(path) → int（统计 ERROR/FATAL/EXCEPTION 行数）

5.4 terminalclaw/tmux_manager.py（Python 实现）
    session_exists(name) → bool
    create_session(name) → None
    kill_session(name) → None
    split_pane(session, direction) → None
    send_keys(session, pane, keys) → None
    set_pane_title(session, pane, title) → None
    attach_session(name) → None
    list_panes(session) → List[Dict]

5.5 terminalclaw/session_store.py（Python 实现）
    读写 .jsonl 格式的会话分析记录
    append_entry(path, record) → None
    read_entries(path) → List[Dict]
    format_entry(model, tokens, analysis, duration) → Dict

5.6 scripts/terminalclaw.sh（Bash 编排脚本）
    入口: case $1 in start|stop|attach|analyze|monitor|status|help
    调用 Python 模块完成各子命令
    颜色输出: ANSI 码

5.7 scripts/tclaw.py（Python CLI 统一入口）
    使用 Click 框架
    命令: start, stop, attach, analyze, monitor, status

5.8 config/ 模板文件
    openclaw.template.json（含 contextTokens + contextWindow）
    Modelfile.terminalclaw
    tmux.conf

5.9 package.json + setup.py
    双入口: tclaw (Node.js) + tclaw.py (pip install)

六、Phase 3 — 测试 + 文档（预计 120 分钟）

6.1 pytest 测试用例
    test_env_check.py: mock 命令存在性、HTTP 响应
    test_log_manager.py: 临时目录文件读写、轮转逻辑
    test_tmux_manager.py: mock tmux 子进程调用
    test_session_store.py: .jsonl 读写一致性

6.2 PROBLEMS.md: 开发过程中遇到的问题与解决方案
6.3 SUMMARY.md: 实现总结、与设计文档的对应关系
